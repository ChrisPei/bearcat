Overview

  • javaScript 自身没有 module system
  • 早期都是用 script tag
  • 手动，重用性差

Overview

javaScript codes grows, need a module system

a module system

  • 在不同文件编写代码
  • 不同文件可以编写不同模块，模块之间代码不会直接影响
  • 编辑和debug，所见即所得
  • 模块代码可以很方便复用
  • 文件的加载最好可以根据模块依赖来异步加载
  • 代码对项目依赖小，可以方便移植到另外一个项目

Solution

  • CommonJS
  • AMD
  • ES6 module

Solution -- CommonJS

require, exports, package.json

同步的

node.js 环境可以直接通过文件I/O 同步 require

browser 环境则需要先build成bundle，然后建立依赖映射

browserify, webpack

Solution -- CommonJS

优点：

  • 模块系统比较直接，符合程序员思维
  • 使用第三方库异常方便，直接package.json里面配置，然后require("library")即可
  • 通过 package.json 可以很好的解决模块之间依赖由于版本冲突的问题

    • Solution -- CommonJS

    缺点：

  • browser环境必须要有一个build过程，代码编写和debug非常痛苦
  • 直接require, exports方式会与项目本身进行耦合，代码复用性比较差
  • 多页面开发比较痛苦

    • Solution -- AMD

    bearcat -- IoC

    • Inversion of Control (IoC) is a design pattern that addresses a component's dependency resolution, configuration and lifecycle.
    • IoC is best understood through the Hollywood Principle: "Dont's call us, we'll call you".
    • Dependency injection (DI), components do not look up, they provide plain simple configuration metadata enabling the container to resolve dependencies.

    bearcat -- IoC

    Without IoC

    var Engine = require('./engine');
var Wheel = require('./wheel');

var Car = function() {
  this.engine = new Engine();
  this.wheel = new Wheel();
}

Car.prototype.run = function() {
  this.engine.run();
  var res = this.wheel.run();
  console.log('run car...');
  return 'car ' + res;
}

module.exports = Car;

    car have to require engine and wheel, and then instantiate with constructor functions, it is tightly coupled

    bearcat -- IoC

    With IoC

    var Car = function($engine) {
  this.$id = "car";
  this.$engine = $engine;
  this.$wheel = null;
}

Car.prototype.run = function() {
  this.$engine.run();
  var res = this.$wheel.run();
  console.log('run car...');
  return 'car ' + res;
}

module.exports = Car;

    car do not have to know where engine, wheel are from, and how they are being instantiated
